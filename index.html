Description

The project supports the saving of gaze data taken by EyeTribe tracker. The main features are:

Saving the data in .csv file

  1. Gaze location
  2. Mouse location
  3. Information about key and mouse button presses

Making a movie from session

  1. Saving the user screen
  2. Adding the gaze location to the frames
  3. Adding gaze history to the frame
  4. Adding visualization of cursor location and mouse clicks.

Usage

Before starting the application, it's necessary to calibrate the eye tracker using the producer software. After launching the application, user is presented with a small window with a button for starting the recording, as well as possibility to choose the location of output files in the top menu. After starting the recording, the user has the possibility to temporarily pause the recording or stop it. After stopping the recording or exiting the applications the output files appear in specified folder.

Code

The files are organized using the MVC pattern. It's generally possible to understand the class purpose by it's name.

  • DataController - Manages the processing the data received from the eye tracker and saving it in .csv file
  • FileController - Manages more complex operations concerning files
  • ImageEditor - Controlls the drawing of the gazes and mouse cursor on the captured screen
  • KeyLogger - Captures the keyboard and mouse button presses during the recording
  • MovieController - Manages assembling the movie from capured images
  • ScreenCaptureController - Controls the main loop of the application

Unfinished features

On the text-recognition branch we kept the work concerning the checking the line of text which is looked at by the user. The TextBlockMarker class handles the recognition of text lines on the image, and although not working perfectly it gives quite good results. Then in the Test class we examined the usability of Tesseract library for optical text recognition. The final results aren't good enough. If we were to work on the project farther, going through the complicated process of training the Tesseract library would be a good start.
